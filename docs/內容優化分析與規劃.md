# iHealthy 網站內容優化：分析報告與規劃建議

## 一、專案與網站概況

### 1.1 技術架構
- **框架**：Astro（靜態站點）
- **內容來源**：`src/content/blog/`，約 **80+ 篇** MDX 文章
- **結構**：首頁、文章列表（分頁）、分類頁、單篇文章（動態路由 `[...slug].astro`）
- **分類**：飲食營養、疾病症狀、營養補充、美容保養、安全防護、環境健康等

### 1.2 內容定位
- **主題**：健康、飲食、營養、疾病預防與自我照護
- **風格**：結合臨床／科學文獻、表格摘要、實用建議與參考文獻
- **讀者**：一般大眾，偏重「有根據、可操作」的知識

---

## 二、目前內容的優點

1. **結構一致**：多數文章具「快速摘要表格 → 分段說明 → 結論 → 相關文章 → 參考文獻」，利於掃讀與對照。
2. **資訊密度高**：表格、要點、數字（如 PACs 36mg、CAC、AQI）清楚，便於查閱。
3. **內鏈設計**：文內與文末有站內連結，有助延長閱讀與 SEO。
4. **專業度**：引用 Cochrane、文獻註腳，有助建立信任感。

## 三、待優化問題分析

### 3.1 技術與資料問題

| 問題 | 說明 | 影響 |
|------|------|------|
| **斷鏈** | 多篇文章連結到 `/sleep-hygiene-and-optimization/`，但站內**無此文章**。涉及：`macronutrients-guide.mdx`、`lifestyle-immunity-factors.mdx`、`is-supplement-works.mdx`。 | 使用者點擊 404，體驗差、內鏈效益喪失。 |
| **錯鏈／錯錨點** | 蔓越莓文末「正確洗手與個人衛生習慣」連到 `/wash-vegetable/`，該文主題為**蔬果清洗**，非洗手。 | 讀者預期與實際內容不符，易混淆。 |
| **自連** | 《心理健康維護》文末「相關文章」含「壓力管理技巧」連到 `/mental_health_maintenance/`（同一篇）。 | 無助導流，顯得未校對。 |
| **檔名錯字** | `lutein-is-suffieciency.mdx`（應為 **sufficiency**）；`how-to-prevent-or-sooth-wrinkles.mdx`（應為 **soothe**）。 | URL 不專業，若已對外分享會造成舊連結失效，需 301 或保留別名。 |

### 3.2 分類與導覽

| 問題 | 說明 |
|------|------|
| **分類標籤不全** | 文章使用的 category：`wellness`、`mental_health`、`mental-health`、`heart-disease`、`health-tech`、`lifestyle` 在 `categories/index.astro` 與 `[category].astro` 的 `categoryLabels` 中**未定義**，分類頁會顯示英文 slug 或 fallback。 |
| **同義分類並存** | `mental_health` 與 `mental-health` 並存，易造成同一主題被拆成兩類。 |

### 3.3 用語與版型一致度

| 項目 | 現況 | 建議 |
|------|------|------|
| **相關區塊標題** | 部分為「相關文章」、部分為「相關閱讀」。 | 統一為一種（建議「相關文章」），全站一致。 |
| **標題層級** | 多數為 H1 → H2 → H3，少數文章 H1 與 frontmatter `title` 不完全一致（如空氣品質文）。 | 約定：正文 H1 與 frontmatter title 一致，或明確規定「H1 用短標、title 用長標」並全站統一。 |
| **結論段** | 有的有「結論」小標，有的直接收尾。 | 建議重要指南類保留「結論」小標，其餘可視長度決定，但同一類型文章風格一致。 |

### 3.4 可讀性與連貫性

- **段落長度**：部分段落較長、一句話含多個概念，掃讀時負擔較大。可適度拆句、或將重點提前成小標/列表。
- **段與段銜接**：少數段落從「機制」跳到「建議」缺少一句過渡（例如「因此，日常可以這樣做：」），補一句即可提升連貫感。
- **專業術語**：首次出現的英文或縮寫（如 PACs、ApoB、CAC、HPA 軸、MBSR）多數有括號說明，可抽樣檢查是否每篇都有一致慣例。
- **受眾一致**：多數為一般讀者，但偶有「審計協定」「進階法」等偏術語的用詞，可視情況改為「復原步驟」「進階飲食」等較口語說法。

### 3.5 內容與時效

- **年份**：部分標題或內文提到「2024 年」（如青春痘文），若希望常駐「最新」建議改成相對說法（如「近年」「目前實證」）或每年檢視更新。
- **參考文獻**：部分為 2024、2023，格式大體一致；可約定是否要加「最後檢索日期」或「更新日」以利讀者判斷時效。

## 四、優化規劃與建議

### 第一階段：修復與統一（優先）

1. **處理斷鏈**
   - **選項 A**：新增一篇「睡眠衛生與優化」文章，slug 設為 `sleep-hygiene-and-optimization`，讓現有連結生效。
   - **選項 B**：若暫不寫該文，將三篇中的睡眠連結改為站內已有文章（若有「生活型態」「壓力」相關文）或暫時移除/改為純文字說明。

2. **修正錯鏈與自連**
   - 蔓越莓文：將「正確洗手與個人衛生習慣」改為連到真正講洗手/衛生的文章，或改為「正確清洗蔬果」並連結 `/wash-vegetable/`。
   - 心理健康文：從「相關文章」中移除連到自己的連結，改為其他 3–4 篇真正相關文章。

3. **統一「相關」區塊標題**
   - 將所有「相關閱讀」改為「相關文章」（或反過來），在 MDX 中批次替換即可。

4. **補齊分類標籤**
   - 在 `categories/index.astro` 與 `[category].astro` 的 `categoryLabels` 中補上：  
     `wellness`、`mental_health`、`mental-health`、`heart-disease`、`health-tech`、`lifestyle` 的中文名稱與 emoji。
   - 將 `mental_health` 與 `mental-health` 統一為同一 key（建議 `mental-health`），並在 MDX frontmatter 中統一使用。

5. **檔名錯字**
   - **方案 A**：重新命名檔案為 `lutein-is-sufficiency.mdx`、`how-to-prevent-or-soothe-wrinkles.mdx`，並在 Astro 或主機層設定 301 從舊 URL 導向新 URL。
   - **方案 B**：若暫不改 URL，至少在 frontmatter 的 title/description 中不使用錯字，並在排程中列入「日後改檔名+301」。

### 第二階段：可讀與連貫

6. **建立簡短寫作規範**（可放在 `docs/` 或 `.cursor/rules`）✅ 已完成
   - 標題：H1 與 frontmatter title 的對應規則。
   - 結論：何時加「結論」小標。
   - 相關區塊：一律「相關文章」、建議 4 篇、不連到自己。
   - 內鏈：錨點文字需與目標文章主題一致，避免「洗手」連到「洗菜」。
   - **產出**：`.cursor/rules/ihealthy-content.mdc`（Cursor 規則）、`docs/寫作規範.md`（完整版）。

7. **抽樣潤稿**
   - 挑 5–10 篇高流量或代表文章：長段拆句、補 1–2 句過渡、術語首次出現加括號、口語化過度術語（如「審計協定」→「復原步驟」）。

8. **時效與年份**
   - 標題或內文中的「2024」改為「近年」或「目前實證」，或建立「年度檢視」清單，每年更新年份與關鍵數據。

### 第三階段：結構與體驗（可選）

9. **導讀與摘要**
   - 若文章超過 1500 字，可考慮在開頭加 2–3 句「本文你會學到……」或 bullet 導讀，方便快速判斷是否要繼續讀。

10. **相關文章品質**
    - 定期檢查：相關文章是否真的與主題相關、是否仍存在、錨點是否仍合適。

11. **SEO 與導覽**
    - 確認每篇的 `description` 與標題能反映內容，且與內文第一段呼應；必要時為長文增加「本頁大綱」錨點連結。
